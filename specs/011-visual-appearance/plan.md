# Implementation Plan: Visual Appearance Improvements

**Branch**: `011-visual-appearance` | **Date**: 2026-04-12 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/011-visual-appearance/spec.md`

## Summary

Three independent CSS-only improvements to the weekly calendar view: (1) swap the visual prominence of the ticket line and the time line in event cards so the ticket is the dominant text element; (2) increase slot row height for more breathing room; (3) change alternating row banding from per-30-minutes to per-hour. All changes are confined to `css/style.css`. No JavaScript, API, or data model changes are needed.

## Technical Context

**Language/Version**: CSS3, JavaScript ES2022 (no changes to JS)

## Implementation Details

### US1 — Event card hierarchy swap (`css/style.css`)

Current rules:
```css
.ev-time    { font-size: 0.75rem; font-weight: 600; white-space: nowrap; overflow: hidden; }
.ev-issue   { font-size: 0.75rem; overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }
.ev-project { font-size: 0.7rem; opacity: 0.75; overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }
```

Target rules (swap prominence; project unchanged):
```css
.ev-time    { font-size: 0.7rem; opacity: 0.75; white-space: nowrap; overflow: hidden; }
.ev-issue   { font-size: 0.75rem; font-weight: 600; overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }
.ev-project { font-size: 0.7rem; opacity: 0.75; overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }
```
