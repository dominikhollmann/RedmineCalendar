# Implementation Plan: Audio Input for AI Assistant (Speech-to-Text)

**Branch**: `021-audio-input-assistant` | **Date**: 2026-04-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `.specify/features/021-audio-input-assistant/spec.md`

## Summary

Add a microphone button to the AI assistant chat panel that uses the browser's Web Speech API (`SpeechRecognition`) for live speech-to-text transcription. When the user stops recording, the transcribed text is auto-sent as a chat message. Includes visual recording feedback, error handling, privacy notice, and i18n support.

## Technical Context

**Language/Version**: JavaScript ES2022 (vanilla, no transpilation)
**Primary Dependencies**: Web Speech API (browser-native), FullCalendar v6 (existing, unchanged)
**Storage**: localStorage — key `redmine_calendar_voice_privacy_dismissed` (boolean)
**Testing**: Vitest (unit), Playwright (UI)
**Target Platform**: Modern desktop and mobile browsers (Chrome, Edge, Safari — Firefox lacks SpeechRecognition support)
**Project Type**: Web application (static SPA)
**Performance Goals**: Live interim results with < 300ms perceived latency; auto-send within 1s of stop
**Constraints**: No external APIs or dependencies; browser-native only; max 60s recording duration
**Scale/Scope**: Single-user tool; chat panel only

## Design Decisions

### Web Speech API Usage

The `SpeechRecognition` (or `webkitSpeechRecognition`) API is used directly:
- `interimResults: true` for live transcription
- `continuous: true` to keep listening until explicit stop or timeout
- `lang` set from app locale (`en-US` or `de-DE`)
- Single recognition session per recording (no streaming chunks)

### Auto-Send Flow

1. User taps mic → privacy notice (first time only) → recording starts
2. Interim results update input field in real time (styled distinctly)
3. User taps stop (or silence/max timeout triggers) → final result replaces interim
4. If input field had existing text, append with space separator
5. Auto-call `handleSend()` from `chatbot.js` — no manual send required
6. Data-changing actions (time entries) still require modal confirmation downstream

### State Machine

```
idle → recording → transcribing → idle
  ↘      ↓              ↓
   error ←──────────────←
```

Transitions:
- `idle → recording`: User taps mic (privacy check passes, permission granted)
- `recording → transcribing`: User taps stop, silence timeout, or max duration
- `transcribing → idle`: Final result received, auto-sent
- `any → error`: Permission denied, no speech, unsupported browser, network failure
- `error → idle`: Error dismissed (auto-clear after display)

### Feature Detection

```javascript
const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;
```

If `SpeechRecognition` is `undefined`, the mic button is hidden entirely (FR-011).
