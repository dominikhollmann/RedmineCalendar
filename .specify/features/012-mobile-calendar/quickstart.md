# Quickstart & Acceptance Tests: Mobile Calendar View

## Acceptance Tests

### US1 — View Calendar on Mobile

- [ ] Open app in mobile browser (or dev tools, 375px width) — calendar renders without horizontal scrolling
- [ ] Verify default view is single-day timegrid
- [ ] Swipe left/right to navigate between days
- [ ] Verify time entries show ticket number, subject, and duration in readable text
- [ ] Rotate to landscape — verify layout adapts

### US2 — Create and Edit Time Entries on Mobile

- [ ] Tap an empty time slot — form opens with touch-friendly inputs
- [ ] Fill in form and save — entry appears on calendar
- [ ] Tap an existing entry — edit form opens with pre-filled values
- [ ] Delete an entry — confirm dialog works on mobile

### US3 — AI Chat on Mobile

- [ ] Open AI chat on mobile — panel fills the screen
- [ ] Ask a question — response is readable
- [ ] AI creates an entry — modal opens in mobile-optimized layout

### Other Features on Mobile

- [ ] Settings page usable on mobile
- [ ] Working hours toggle works
- [ ] Docs panel opens full-screen on mobile
